# Narrator

AI-powered desktop app that generates narration scripts from videos. Built with Tauri v2 (Rust backend) + React 19 (TypeScript frontend).

## Architecture

- **Backend:** `src-tauri/src/` — Rust with Tauri v2, Tokio async runtime
  - `commands.rs` — IPC command handlers (the main API surface)
  - `ai_client.rs` — Claude / OpenAI / Gemini API orchestration
  - `elevenlabs_client.rs` — ElevenLabs TTS
  - `azure_tts_client.rs` — Azure TTS
  - `video_engine.rs` — ffmpeg frame extraction & probing
  - `doc_processor.rs` — PDF/Markdown/TXT text extraction
  - `export_engine.rs` — SRT/VTT/JSON/Markdown/SSML export
  - `project_store.rs` — Project persistence at `~/.narrator/`
  - `telemetry.rs` — Aptabase analytics (EU-hosted, no PII)
  - `models.rs` — Shared types
  - `menu.rs` — Native menu (macOS/Windows/Linux)
- **Frontend:** `src/` — React 19 + Zustand + Tailwind CSS 4
  - `src/features/` — Feature modules (wizard steps, settings, legal, telemetry)
  - `src/stores/` — Zustand state stores
  - `src/components/` — Shared UI components
  - `src/lib/tauri/commands.ts` — Typed IPC wrappers for all Tauri commands
- **Config:** `src-tauri/tauri.conf.json` — App config, CSP, updater, bundle settings
- **Capabilities:** `src-tauri/capabilities/default.json` — Tauri v2 permission ACL

## Commands

```bash
pnpm install              # Install frontend deps
pnpm dev                  # Vite dev server (frontend only)
pnpm tauri dev            # Full app dev mode (frontend + Rust)
pnpm build                # TypeScript check + Vite build
pnpm typecheck            # TypeScript type check only
pnpm test                 # Run unit tests (vitest)
cargo test --manifest-path src-tauri/Cargo.toml   # Rust tests
cargo clippy --manifest-path src-tauri/Cargo.toml -- -D warnings  # Rust lint
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check         # Rust format check
```

## Quality Gates

PR quality gate (`.github/workflows/pr-quality-gate.yml`) runs on every PR to `main`:
1. Version consistency check across `package.json`, `tauri.conf.json`, `Cargo.toml`
2. Frontend: `pnpm typecheck` + `pnpm test` (vitest with coverage) + `pnpm build`
3. Rust: `cargo fmt --check` + `cargo clippy -D warnings` + `cargo test`

All three must pass before merge.

## Testing

- **Frontend tests:** `src/__tests__/` — Vitest + React Testing Library + jsdom
  - Test setup in `src/__tests__/setup.ts` — mocks all Tauri IPC commands
  - Use `setupDefaultMocks()` and `resetAllStores()` in `beforeEach`
  - Mock IPC with `mockIPC` from `@tauri-apps/api/mocks`
- **Rust tests:** Standard `#[cfg(test)]` modules, run with `cargo test`
- **Store tests:** `src/stores/*.test.ts` for Zustand store logic

## Version Management

Version must be kept in sync across three files:
- `package.json` → `version`
- `src-tauri/tauri.conf.json` → `version`
- `src-tauri/Cargo.toml` → `version`

The PR quality gate enforces this.

## Releases

- Release workflow triggers on `v*` tags
- Builds for macOS (Apple Silicon + Intel) and Windows
- Creates a **draft** release — publish manually with `/release` skill
- Auto-update requires `.sig` files and `latest.json` in the release (generated by tauri-action when signing keys are configured)

## Key Conventions

- Frontend uses inline styles (not CSS modules) — match existing patterns
- Design tokens in `src/lib/theme.ts` (`colors`, `typography`, `spacing`)
- Settings panel uses local `C` object for colors
- All Tauri commands are wrapped in `src/lib/tauri/commands.ts` — never call `invoke()` directly from components
- Telemetry events are fire-and-forget via `trackEvent()` from `src/features/telemetry/analytics.ts`
- API keys stored locally at `~/.narrator/config.json` with 0600 permissions
- No PII in telemetry — never track file paths, content, or user identifiers
